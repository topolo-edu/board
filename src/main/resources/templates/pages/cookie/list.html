🍪 httpOnly Cookie 방식의 특징

✅ 장점

  • • JavaScript에서 토큰에 접근할 수 없습니다
  • • XSS 공격으로부터 토큰을 보호합니다
  • • 브라우저에서 자동으로 쿠키를 전송합니다
  • • 브라우저 종료 시까지 유지됩니다

⚠️ 주의사항

  • • CSRF 공격에 취약할 수 있습니다
  • • 서버에서 CSRF 토큰 구현이 필요합니다
  • • SameSite 설정으로 보안 강화 가능합니다
  • • 서버 설정이 복잡할 수 있습니다

🔒 JavaScript 접근 불가 체험

httpOnly Cookie의 보안 특성을 체험해보세요:

  1. 브라우저 개발자 도구를 열어보세요 (F12)
  2. Console 탭에서 document.cookie를 입력해보세요
  3. JWT 토큰이 보이지 않는 것을 확인하세요
  4. Application 탭 > Cookies에서만 확인 가능합니다

💡 XSS 공격으로부터 안전한 이유입니다!

⚠️ CSRF 공격 주의사항

httpOnly Cookie 방식의 CSRF 취약점:

  • 악성 사이트에서 사용자의 쿠키를 이용한 요청 가능
  • SameSite=Strict 설정으로 완화 가능
  • CSRF 토큰 구현으로 추가 보안 가능
  • 실제 운영에서는 반드시 CSRF 대책 필요

💡 현재는 교육용으로 CSRF 보호가 비활성화되어 있습니다.
