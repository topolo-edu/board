🍪 httpOnly Cookie 글쓰기

httpOnly Cookie에 저장된 JWT 토큰으로 인증하여 글을 작성합니다

최대 200자까지 입력 가능합니다.

최대 5,000자까지 입력 가능합니다. httpOnly Cookie는 자동 저장되지 않습니다.

🔄 자동 토큰 갱신 체험

httpOnly Cookie 방식의 토큰 갱신 과정:

  1. 브라우저가 자동으로 httpOnly Cookie를 요청 헤더에 포함
  2. API 호출 시 401 에러 발생하면 자동으로 Refresh Token Cookie 사용
  3. 서버에서 새로운 httpOnly Cookie 설정
  4. 실패한 요청을 새 쿠키로 재시도

💡 "토큰 상태 확인" 버튼을 눌러보세요!

🍪 httpOnly Cookie 방식 특징

구현 방식

  • • 서버에서 Set-Cookie 헤더로 설정
  • • HttpOnly 플래그로 JavaScript 접근 차단
  • • 브라우저가 자동으로 쿠키 전송

보안 고려사항

  • • JavaScript에서 토큰 접근 불가
  • • XSS 공격으로부터 안전
  • • CSRF 공격에 취약할 수 있음
  • • SameSite 설정으로 보안 강화 가능

🔒 JavaScript 접근 불가 체험

httpOnly Cookie의 보안 특성을 직접 확인해보세요:

  1. 브라우저 개발자 도구를 열어보세요 (F12)
  2. Console 탭에서 document.cookie를 입력해보세요
  3. JWT 토큰이 표시되지 않는 것을 확인하세요
  4. Application 탭 > Cookies에서는 확인할 수 있습니다
  5. JavaScript로는 토큰을 읽거나 수정할 수 없습니다

💡 이것이 XSS 공격을 방어하는 핵심 메커니즘입니다!

⚠️ CSRF 공격 취약점

httpOnly Cookie 방식의 CSRF 취약점을 이해하세요:

  • 브라우저가 자동으로 쿠키를 전송하기 때문에 CSRF 공격 가능
  • 악성 사이트에서 사용자 모르게 요청을 보낼 수 있음
  • SameSite=Strict 설정으로 완화 가능
  • CSRF 토큰 구현으로 추가 보안 강화 가능

💡 실제 운영에서는 반드시 CSRF 보호 메커니즘이 필요합니다!
